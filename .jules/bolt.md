## 2026-04-22 - Redundant Base64 encoding and String-to-Bytes conversion in `StringToBase64Stream`
**Learning:** Found a specific anti-pattern in Pascal/Lazarus codebase where string parameters were being unnecessarily converted to bytes and back to strings using `TEncoding.UTF8.GetBytes` and `GetString`, right before passing into functions that natively accepted string inputs (like `base64.EncodeStringBase64`). Furthermore, calling the same function twice sequentially as arguments to a method (`WriteBuffer(Encode(...)[1], Length(Encode(...)))`) causes the compiler to evaluate it redundantly.
**Action:** Avoid string-to-bytes-to-string round-trips for base64 encoding when native string inputs are supported. Always store the result of expensive base64 encodings in a local variable instead of recalculating it for both value indexing and length determination. Always ensure 1-based Pascal strings have `Length > 0` before accessing index `[1]`.
